# 📖 คู่มือการติดตั้งและเรียกใช้งานระบบทดสอบ (Run Guide)

คู่มือนี้สำหรับระบุขั้นตอนการติดตั้งและรันคำสั่งต่าง ๆ ของระบบ **Chatbot QA Automation** ตั้งแต่เริ่มต้น

---

## 🛠️ ขั้นตอนที่ 1: การเตรียมการก่อนเริ่มรัน (Prerequisites)

1. ติดตั้ง **Node.js** (เวอร์ชันแนะนำคือ 18.0.0 ขึ้นไป)
2. หากจำเป็นต้องรันผ่านเครือข่ายภายใน ให้ตรวจสอบการเชื่อมต่อ **VPN** ให้เรียบร้อย
3. ตรวจสอบว่ามีไฟล์คีย์บัญชีบริการของ Google ในโปรเจกต์ เช่น:
   - `credentials/chatbot-497504-0836c2cb62a2.json`
4. ตรวจสอบการตั้งค่าในไฟล์ `.env` ว่ามี `GOOGLE_DOCUMENT_ID` และ `GOOGLE_SERVICE_ACCOUNT_KEY_PATH` ตรงตามที่ใช้งาน

---

## 📥 ขั้นตอนที่ 2: ติดตั้ง Dependencies (ครั้งแรกเท่านั้น)

เปิด Terminal ในโฟลเดอร์โปรเจกต์นี้ แล้วรันคำสั่งดังต่อไปนี้ตามลำดับ:

### 1. ติดตั้งไลบรารี Node.js
```bash
npm install
```

### 2. ติดตั้งเบราว์เซอร์ Playwright (Chromium)
```bash
npm run install:browsers
# หรือ npx playwright install chromium
```

---

## ⚙️ ขั้นตอนที่ 3: ตั้งค่าการรันแถวทดสอบ (Optional)

คุณสามารถใช้โปรแกรมจัดการค่าการรัน `run_config.js` ที่เพิ่มเข้ามาเพื่อกำหนดช่วงแถวและทิศทางการรันได้:

* **เปิดเมนูตั้งค่าแบบ Interactive (แนะนำ):**
  ```bash
  node run_config.js
  ```
  *(ระบบจะให้เลือกช่วงเริ่มต้น-สิ้นสุด, เลือกทิศทางการรัน เช่น บนลงล่าง/ล่างขึ้นบน/แบ่งครึ่ง, และเลือกเลขคู่/คี่)*

* **แสดงการตั้งค่าปัจจุบัน:**
  ```bash
  node run_config.js --show
  ```

* **รีเซ็ตการตั้งค่ากลับเป็นค่าเริ่มต้น (รันทุกแถวจากบนลงล่าง):**
  ```bash
  node run_config.js --reset
  ```

---

## ▶️ ขั้นตอนที่ 4: คำสั่งการเรียกใช้งานการทดสอบ (Running Tests)

การรันจะแบ่งออกเป็น 2 โหมดหลักตามเป้าหมายปลายทาง (Google Docs หรือ Google Sheets):

### 1. กรณีรันบันทึกผลลง Google Docs (ค่าเริ่มต้น)

* **รันการทดสอบทั้งหมด (ตามที่ตั้งค่าใน run_config):**
  ```bash
  npm test
  ```
* **รันเพื่อทดสอบเบื้องต้น (Dry Run — ไม่บันทึกผลกลับ Google Docs):**
  ```bash
  npm run test:dry
  ```
* **รันต่อจากจุดที่ค้างอยู่ (Resume — ข้ามข้อที่ผ่านไปแล้วใน checkpoint):**
  ```bash
  npm run test:resume
  ```
* **เริ่มรันใหม่ทั้งหมดโดยล้างประวัติ (Reset Checkpoint):**
  ```bash
  npm run test:reset
  ```
* **เขียนผลลัพธ์จาก Checkpoint ลง Docs อย่างเดียว (ข้ามการรันเบราว์เซอร์):**
  ```bash
  npm run test:docs-only
  ```

---

### 2. กรณีรันบันทึกผลลง Google Sheets
*(ต้องเอาเครื่องหมาย `#` ออกหน้า `GOOGLE_SPREADSHEET_ID` ใน `.env` ก่อนใช้งาน)*

* **รันการทดสอบบันทึกผลเข้า Sheets:**
  ```bash
  npm run test:sheets
  ```
* **รันแบบไม่บันทึกผล (Dry Run — Sheets):**
  ```bash
  npm run test:sheets:dry
  ```
* **รันต่อจากจุดค้างเดิมบันทึกผลเข้า Sheets:**
  ```bash
  npm run test:sheets:resume
  ```
* **เริ่มรันใหม่ล้างประวัติของ Sheets:**
  ```bash
  npm run test:sheets:reset
  ```
* **เขียนผลลัพธ์จาก Checkpoint ลง Sheets อย่างเดียว (ข้ามการรันเบราว์เซอร์):**
  ```bash
  npm run test:sheets-only
  ```

---

## 📊 ตารางสรุปการวิเคราะห์ผลลัพธ์ (PASS/PARTIAL/FAIL)

ระบบจะเปรียบเทียบคำตอบของ Chatbot กับผลลัพธ์ที่คาดหวังใน Google Docs/Sheets ตามเกณฑ์ดังนี้:

| ผลลัพธ์ | สีที่แสดง | เงื่อนไข |
| :--- | :---: | :--- |
| **PASS** | 🟢 เขียว | ความคล้ายคลึงของข้อความ (Similarity) $\ge$ 65% |
| **PARTIAL** | 🟡 เหลือง | ความคล้ายคลึงอยู่ระหว่าง 30% ถึง 65% (จะทำการแคปเจอร์หน้าจอเก็บใน `screenshots/partial/`) |
| **FAIL** | 🔴 แดง | ความคล้ายคลึง < 30% หรือตรวจเจอคำผิดพลาด เช่น "ไม่มีข้อมูล" หรือเกิด Timeout |
